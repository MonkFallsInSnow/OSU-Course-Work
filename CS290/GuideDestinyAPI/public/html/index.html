The Destiny API

A basic how-to guide

What this guide hopes to accomplish

Welcome! This basic how-to guide aims to set you up with all the information you will need to make public get requests using the Destiny API. By the end of this guide, you should be able to get a variety of interesting data about your Destiny characters, and, since all the API get requests are relatively similar, with a little reading from the documentation, you should be prepared to grab any other information you may want. What this guide will not cover are post requests and private endpoints. The documentation on those is sparse and generally requires login credentials including the X-CSRF HTTP header and required token.



What you need to know

Since Bungie does not currently support, "Requests from JavaScript hosted by another web site...", the requests we will be making involve server side JavaScript, so some knowledge of the language will go a long way. In doing so, if you would like to run some of the code examples yourself, you will need a basic understanding of how to work with node.js including the express.js, request.js, body-parser.js and handlebars.js modules. You can find links to the documentation for each of these modules on the resource page.



What is the Destiny API?

Put simply, the Destiny API is a service that allows one to make server side get and post requests in order to collect a vast amount of data about game characters, in-game activities, vendors, equipment stats, user accounts and forum posts. With this information, you will be able to create web and mobile apps that use this content in useful and interesting ways. On the next page of this guide, you will find a very basic app that queries the API for a list of equipment currently used by a specific character. All you need do is enter you PSN or Xbox Live username, select your account platform and which character’s items you want to view. If you don’t have an account, just click the “No Account” radio button and a table displaying my own character’s equipment will present itself. Let’s get started.
